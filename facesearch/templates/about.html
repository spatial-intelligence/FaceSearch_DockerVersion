Details:

This tool is designed to assist in searching for a person's face in an image collection.

The software scans each image for faces then generates a corresponding face encoding file. 

This can take a while - therefore there are 2 search facilities offered: 


    Option 1) HOG mode: +fast -accuracy

    Option 2) CNN mode: +accuracy -slow


Once a face encoding has been computed it is saved to disk with the extension .fe_hog or .fe_cnn.

These encodings will be used in the future so that repeat searches of the same image collection will be much faster.


Notes:
  • The tool should be considered as an assistant and not replacement for human checkers. 
  • The tool will return false positives and false negatives - the results are ranked with best match first. 
  • Dlib is known to have issues matching children's faces and Asian faces, due to the dataset it was trained on.
  • The face likeness score is between 0.0 (same) and 1.0 and is exponential. So the likeness difference between a score 0.2 to 0.3 is much smaller than the difference between a 0.5 and 0.6. 



How to use:

Put the files you wish to scan into a directory on your PC which is shared with the Docker container as 'data' Image files to be scanned must be placed in a folder structure as follows: 


    /osr4rights/search - files to be searched to find the target face(s)

    /osr4rights/target - files which define the target face(s) to find


Image files must be of type .jpg or .png 


Set the target face(s) to find. This can be several different pictures of the same person. 

The software will then search for any matches. 

The output report will rank from close matches to distant matches. 

Please note this can take a long while depending on the speed of the PC and number of files to scan. 



Options:

There is a toggle for scanning small faces. This enlarges the original image and divides into 4 sections before scanning for faces. This is done to keep the memory usage as low as possible, while giving better recognition for smaller faces in images. However the extra image processing will increase the processing time.


Use the image details page to check how many faces were found in each image, and check a sample to see if you need to enable the smallfaces option. It can over-report the number of faces found in the image details list if small faces scanning is included, due to the double processing of each image.
